What is Grunt

  • Grunt is a JavaScript task runner.
    • Unit Tests
    • CSS Pre-processors
    • JS/CSS Hinting
    • File Manipulation
    • File Watching
  • Runs on top of NodeJS.
  • Plugins for everything.

Installing Grunt

There are two parts to Grunt:

  • grunt-cli
  • plugins

To install the command line interface:

npm install grunt-cli -g

Plugins

Plugins

Plugins are the things that tell Grunt to do "stuff".

www.gruntjs.com/plugins

"contrib" plugins are the "official" plugins.

Installing Plugins

Each plugin page will have install instructions, but typically it's npm install <pluginname>:
npm install grunt-contrib-uglify --save-dev
--save-dev modifies your package.json file (more on this soon)

Configuring Grunt

package.json

  • Node.JS convention (not grunt)
  • Stores what plugins your project uses
  • Not required bug strongly recommended.

package.json

{
  "name": "Grunt_Presentation",
  "version": "2013.7.25",
  "devDependencies": {
    "grunt": "~0.4.1",
    "grunt-contrib-uglify": "~0.2.0",
    "grunt-contrib-concat": "~0.1.3"
  }
}

Remember --save-dev from a few slides back? That makes sure entries are added under the "devDependencies" node.

Tasks

  • Tasks are direct children of the Gruntfile.js object.
  • They are always the same name as the plugin name. E.G. concat , uglify
  • The plugins for your Tasks MUST be registred in your Gruntfile by calling grunt.initTask( '<taskname>');

Tasks

{
    concat: {
        ...
    }
    uglify: {
        ...
    },
    //don't forget to load your plugins!
    grunt.loadNpmTasks( 'concat' );
    grunt.loadNpmTasks( 'uglify' );
}

Custom Tasks

{
    ...
    
    grunt.registerTask( 'build', 
        ['concat', 'uglify' ] );
        
    ...
}

Running Tasks

Run all

$ grunt

Run one

$ grunt concat
$ grunt build

Targets

Targets are sub-configurations of tasks that can be "targeted" when running a task.

    {
        copy : {
            dev : {
                //path to dev
            },
            staging : {
                //path to staging
            },
            production: {
                //path to prod
            }
        }
    }

Configuring Tasks
