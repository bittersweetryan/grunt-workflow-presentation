Installing Grunt

Installing Grunt

There are two parts to Grunt:

  • grunt-cli (global command line interface)
  • grunt task runner (project local task runner)

Installing the CLI

npm install -g grunt-cli

Installing Local Grunt

Local Grunt should be installed in each projects root folder.

npm install -g grunt-cli

Configure Grunt For Your Project

  1. package.json
  2. Install Plugins

package.json

  • Project Name
  • Current Version
  • Dependencies (Grunt plugins)

package.json

{
  "name": "ember-model",
  "description": "A lightweight model library for Ember.js",
  "main": "Gruntfile.js",
  "devDependencies": {
      ...
  },
  "version" : "0.2.1"
}

Without plugins Grunt doesn't do much!

Plugins are what Grunt uses to do "stuff"

www.gruntjs.com/plugins

"contrib" plugins are the "official" plugins.

Installing Plugins

Plugins are installed with the npm install command

npm install grunt-contrib-uglify --save-dev

--save-dev modifies your package.json file

Configuring Grunt

Each project needs a Grunt configuration.

package.json

  • Node.JS convention (not grunt)
  • Stores what plugins your project uses
  • Not required bug strongly recommended.

package.json

{
  "name": "Grunt_Presentation",
  "version": "2013.7.25",
  "devDependencies": {
	"grunt": "~0.4.1",
	"grunt-contrib-uglify": "~0.2.0",
	"grunt-contrib-concat": "~0.1.3"
  }
}

Remember --save-dev from a few slides back? That makes sure entries are added under the "devDependencies" node.

Gruntfile.js

4 major pieces

  • Tasks
  • Targets
  • Files
  • Options

Sample Gruntfile.js

module.exports = function( grunt ){
 
	var sourceFiles = [ 's-code.js', 's-code-page.js'];
 
	grunt.initConfig({
		pkg : grunt.file.readJSON( 'package.json' ),
 
		uglify : {
			all : {
				files : {
					's-code.min.<%= pkg.version %>.js' : sourceFiles
				}
			}
		},
		jasmine : {
			unmodified : {
				src : sourceFiles,
				options : {
					specs : [ '**test/spec/*.js' ]
				}
			},
			min : {
				src : [ 's-code.min.<% pkg.version %>.js' ],
				options : {
					specs : [ '**test/spec/*.js' ]
				}
			}
		},
		clean : {
			build : {
				src : [ 's-code.min**.js' ]
			}
		}
	});
 
	//dynamically load tasks
	require( 'matchdep' ).filterDev( 'grunt-*' ).forEach( grunt.loadNpmTasks );
 
	grunt.registerTask( 'default', ['jasmine:unmodified:build', 'clean', 'uglify', 'jasmine:min:build']);
};

Tasks

  • Tasks are direct children of the Gruntfile.js object.
  • They are always the same name as the plugin name. E.G. concat , uglify
  • The plugins for your Tasks MUST be registred in your Gruntfile by calling grunt.initTask( '<taskname>');

Tasks

{
	concat: {
		...
	}
	uglify: {
		...
	},
	//don't forget to load your plugins!
	grunt.loadNpmTasks( 'concat' );
	grunt.loadNpmTasks( 'uglify' );
}

Custom Tasks

{
	...
	
	grunt.registerTask( 'build', 
		['concat', 'uglify' ] );
		
	...
}

Running Tasks

Run all

$ grunt

Run one

$ grunt concat
$ grunt build

Targets

Targets are sub-configurations of tasks that can be "targeted" when running a task.

	{
		copy : {
			dev : {
				//path to dev
			},
			staging : {
				//path to staging
			},
			production: {
				//path to prod
			}
		}
	}

$ grunt copy:dev

Configuring Tasks

  • Tasks also have optional "options" that can be passed into them
